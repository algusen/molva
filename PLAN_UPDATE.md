# PLAN_UPDATE.md — переход daemon-first → CLI-first

**База (что у агента сейчас):** `PLAN.md` — Pisar, «тёплый демон + триггер из Finder», фазы 0–5.
**Цель:** `PLAN.md` — Pisar, «CLI-first», фазы 0–7. Новая версия — source of truth,
она заменяет старый файл целиком. Ниже — дельта для сверки, чтобы не потерять контекст.

**Причина:** CLI и демон — два фронтенда над одним `service.py`. CLI-first упрощает отладку
(standalone без демона и модели) и позволяет проверять реальную модель прямо из терминала.
Всё ядро Phase 1 переиспользуется без изменений.

### Роль основного интерфейса (в шапку)
```diff
+Итоговый запуск — правым кликом в Finder; но основной интерфейс и тест-харнесс —

```

### Ключевые архитектурные решения (добавлено)
```diff
+- Единое ядро `service.transcribe(...)`; все фронтенды (CLI, демон, Finder) вызывают только его.
+- CLI в стиле ffmpeg: вход/выход позиционно + флаги; вывод в stdout для pipe.
+- Два режима CLI:
+    standalone — пайплайн в процессе (тесты; холодная загрузка модели);
+    client     — форвардит задачу демону (тёплая модель).
+  Нет демона → standalone; появился → client по умолчанию с фолбэком.
```

### Новый блок «Интерфейс CLI» (добавлен целиком, проектируется в Фазе 2)
```diff
+    pisar [ОПЦИИ] <input>...
+    -f/--format txt|srt|vtt|json   -o/--output PATH   -l/--lang ru
+    --backend stub|gigaam   --revision e2e_rnnt|e2e_ctc   --device auto|cpu|mps
+    --vad on|off   --stdout   --json   --overwrite   -v/--verbose
+    --no-daemon/--daemon
+    pisar daemon start|stop|status   pisar health
```
