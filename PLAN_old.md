# PLAN.md — Pisar (Вариант 2: тёплый демон + триггер из Finder)

Проект **Pisar** — локальное приложение для транскрибации аудио/видео на macOS
(Apple Silicon) на базе модели `ai-sage/GigaAM-v3`. Запуск — правым кликом по файлу в Finder.

Соглашение об именах: Python-пакет — `pisar`; launchd-метка — `com.<user>.pisar`;
пункт в Finder — «Pisar».
Демон держит модель в памяти, клик лишь передаёт ему путь к файлу.

Этот файл — рабочий план. Шаги передаются исполнителю (мне) по одному.
Каждый шаг имеет критерий приёмки; пока он не выполнен, к следующему не переходим.

---

## Модель исполнения

- **Контейнер (Linux)** — здесь собирается и тестируется ядро: логика демона, HTTP API,
  препроцессинг ffmpeg, нарезка по тишине (VAD), форматы вывода, инсталлятор,
  .workflow-бандл для Finder. Тесты гоняются на `StubTranscriber` (заглушке).
- **Mac (Apple Silicon)** — здесь загружается настоящая модель GigaAM-v3, ставится
  `launchd`-агент, регистрируется Quick Action, проходят end-to-end проверки.
- Обозначения проверок: `[контейнер]` — проверяю сам; `[Mac]` — запускаете вы.

## Ключевые архитектурные решения (фиксируются на старте)

- Демон слушает только `127.0.0.1` — всё локально, наружу ничего не уходит.
- Транспорт — HTTP (FastAPI). Передаётся **путь** к файлу, сам файл остаётся на диске.
  REST-API сразу станет точкой интеграции для будущего варианта 3 (SwiftUI-приложение).
- Модель грузится один раз при старте демона; `/health` сообщает готовность.
- Инференс сериализуется (один процесс, одна модель) — очередь/лок, чтобы не словить OOM.
- Бэкенд транскрайбера выбирается через конфиг: `stub` (тесты) / `gigaam` (Mac).
- Вывод — sidecar-файлы рядом с исходником (`.txt`, `.srt`, `.vtt`) + уведомление,
  опционально копирование в буфер обмена.
- Язык по умолчанию — русский (GigaAM). Точка расширения под Whisper заложена в API,
  но не реализуется в варианте 2.

## Технологический стек (фиксируется на старте)

- Python 3.11+, виртуальное окружение через `venv` (в инсталляторе), `uv` — опционально.
- FastAPI + uvicorn — демон.
- ffmpeg — извлечение/ресемплинг аудио (16 кГц, mono, WAV).
- silero-vad — нарезка длинных файлов по тишине (лёгкий, без токенов HF).
- GigaAM-v3, ревизия `e2e_rnnt` — даёт пунктуированный нормализованный текст напрямую.
- transformers + torch (рекомендованные версии модели), `trust_remote_code=True`.
- launchd LaunchAgent — автозапуск и перезапуск демона.
- Finder Quick Action как `.workflow`-бандл (генерируется, ставится инсталлятором).
- terminal-notifier (или osascript) — уведомления.

# ФАЗА 1 — Ядро демона (тестируется на заглушке)

### Шаг 1.1 — Интерфейс Transcriber + StubTranscriber
- **Цель:** абстракция движка, позволяющая тестировать всё без модели.
- **Что делаю:** протокол `Transcriber` (`transcribe(wav_path, language) -> list[Segment]`,
  где `Segment = {start, end, text}`); `StubTranscriber`, возвращающий заранее заданные
  сегменты по длине входа.
- **Артефакты:** `src/.../transcriber/base.py`, `.../transcriber/stub.py`, тесты.
- **Приёмка `[контейнер]`:** unit-тест: заглушка возвращает корректные `Segment`-ы;
  типы и тайминги валидны.

### Шаг 1.2 — Препроцессинг аудио (обёртка ffmpeg)
- **Цель:** привести любой аудио/видео-вход к 16 кГц mono WAV.
- **Что делаю:** `audio.py`: `probe()` (длительность, кодеки), `to_wav16k_mono()`,
  управление временными файлами, понятные ошибки если ffmpeg отсутствует.
- **Артефакты:** `src/.../audio.py`, тесты.
- **Приёмка `[контейнер]`:** ставлю ffmpeg в контейнер, генерирую тестовый тон,
  проверяю что на выходе WAV 16000 Гц / 1 канал; извлечение дорожки из тест-видео работает.

### Шаг 1.3 — Нарезка длинных файлов (VAD)
- **Цель:** резать длинные записи на сегменты речи по тишине.
- **Что делаю:** `vad.py` на silero-vad: вход — wav, выход — список интервалов речи
  с настраиваемыми порогами и склейкой коротких пауз; fallback на нарезку фиксированными
  окнами с перекрытием, если VAD недоступен.
- **Артефакты:** `src/.../vad.py`, тесты.
- **Приёмка `[контейнер]`:** на синтетическом «речь–тишина–речь» wav возвращаются
  ожидаемые границы; пустой/короткий вход обрабатывается без падения.

### Шаг 1.4 — HTTP API демона
- **Цель:** рабочий сервис, держащий модель в памяти.
- **Что делаю:** FastAPI-приложение: `GET /health` (статус загрузки модели),
  `POST /transcribe` (тело: `path`, `formats`, `language`); сборка пайплайна
  препроцессинг → VAD → транскрайбер → сегменты; глобальный лок на инференс
  (сериализация запросов); аккуратные коды ошибок (нет файла, не готов, ffmpeg упал).
- **Артефакты:** `src/.../api.py`, `src/.../service.py`, тесты на `TestClient`.
- **Приёмка `[контейнер]`:** с `StubTranscriber` тесты проходят; `/health` отдаёт
  «готов»; два параллельных запроса сериализуются; ошибки дают корректные HTTP-коды.

### Шаг 1.5 — Запись результатов и нотификации
- **Цель:** превратить сегменты в файлы рядом с исходником + сообщить пользователю.
- **Что делаю:** `output.py`: писатели `.txt`/`.srt`/`.vtt` (корректные тайм-коды),
  правила именования sidecar (`<имя>.txt` и т.д., без перезатирания — суффикс при конфликте);
  хуки уведомления и копирования в буфер (no-op в Linux, реальные на macOS).
- **Артефакты:** `src/.../output.py`, тесты.
- **Приёмка `[контейнер]`:** SRT/VTT проходят валидацию формата и тайм-кодов;
  имена sidecar формируются по спецификации из CONTRACT.md.

# ФАЗА 2 — Реальный бэкенд модели (Mac)

### Шаг 2.1 — GigaAMTranscriber
- **Цель:** настоящая транскрибация на GigaAM-v3.
- **Что делаю:** реализация `Transcriber` через transformers (`ai-sage/GigaAM-v3`,
  revision `e2e_rnnt`, `trust_remote_code=True`); выбор устройства (CPU/MPS) с фолбэком;
  прогон каждого VAD-сегмента и сшивка с таймингами.
- **Артефакты:** `src/.../transcriber/gigaam.py`, инструкция по зависимостям модели.
- **Приёмка `[Mac]`:** модель грузится; короткий русский клип транскрибируется;
  текст осмысленный и пунктуированный.

### Шаг 2.2 — Переключение бэкенда через конфиг
- **Цель:** один и тот же демон работает и с заглушкой, и с моделью.
- **Что делаю:** фабрика транскрайбера по `BACKEND=stub|gigaam`; прогрев модели
  при старте; `/health` не «готов» пока модель грузится.
- **Артефакты:** правки `config.py`, `service.py`, точки входа.
- **Приёмка `[Mac]`:** демон стартует с `BACKEND=gigaam`, `/health` доходит до «готов»,
  end-to-end `POST /transcribe` по реальному файлу пишет корректный `.txt`.

# ФАЗА 3 — Управление процессом и триггер

### Шаг 3.1 — launchd LaunchAgent
- **Цель:** демон автозапускается и переживает перезапуск/краши.
- **Что делаю:** `.plist` (RunAtLoad, KeepAlive, рабочая директория, env, пути логов
  stdout/stderr), скрипт запуска демона через uvicorn.
- **Артефакты:** `packaging/com.<user>.pisar.plist`, `scripts/run_daemon.sh`.
- **Приёмка:** `[контейнер]` — `plutil`-эквивалент/валидатор XML проходит;
  `[Mac]` — агент грузится через `launchctl`, демон жив после повторного входа в систему,
  поднимается заново после kill.

### Шаг 3.2 — Тонкий клиент
- **Цель:** маленький скрипт, который клик передаёт демону.
- **Что делаю:** `pisar` (shell или python): принимает путь(и),
  POST на `127.0.0.1`, ждёт/поднимает демон если не готов, по завершении —
  уведомление/вывод; коды возврата для интеграции с Quick Action.
- **Артефакты:** `scripts/pisar`.
- **Приёмка:** `[контейнер]` — `shellcheck` чистый, клиент против тестового демона
  на заглушке отрабатывает; `[Mac]` — транскрибирует реальный файл через живой демон.

### Шаг 3.3 — Quick Action в Finder
- **Цель:** пункт правого клика в Finder.
- **Что делаю:** генерирую `.workflow`-бандл (Info.plist + document.wflow), который
  вызывает тонкий клиент для выбранных файлов; инсталлятор кладёт его в
  `~/Library/Services/`. Альтернатива (Apple Shortcut) — отдельной инструкцией.
- **Артефакты:** `packaging/Pisar.workflow/...`.
- **Приёмка `[Mac]`:** ПКМ по файлу → «Быстрые действия» → запуск → sidecar появляется,
  уведомление приходит.

# ФАЗА 4 — Упаковка и деплой

### Шаг 4.1 — install.sh / uninstall.sh
- **Цель:** установка в одну команду на чистой машине.
- **Что делаю:** `install.sh` — проверка/подсказка по brew-зависимостям (ffmpeg,
  terminal-notifier), создание venv, установка пакета, выгрузка модели (первый прогон),
  установка plist и загрузка агента, регистрация Quick Action; идемпотентность.
  `uninstall.sh` — выгрузка агента, удаление сервиса, очистка.
- **Артефакты:** `install.sh`, `uninstall.sh`.
- **Приёмка:** `[контейнер]` — `shellcheck`, dry-run логики; `[Mac]` — чистая установка
  с нуля, повторный запуск ничего не ломает.

### Шаг 4.2 — README
- **Цель:** инструкция по установке, использованию, диагностике.
- **Что делаю:** README (требования, установка, как пользоваться, выбор формата вывода,
  заметка про русский язык, типичные проблемы, как смотреть логи демона).
- **Артефакты:** `README.md`.
- **Приёмка `[контейнер]`:** ревью на полноту и соответствие реальным командам.

### Шаг 4.3 — End-to-end приёмка
- **Цель:** подтвердить работу на реальном материале.
- **Что делаю:** чек-лист приёмки (матрица: короткое аудио, длинное аудио,
  видео→`.srt`, не тот язык/битый файл — корректная ошибка).
- **Приёмка `[Mac]`:** все пункты матрицы зелёные; задержка отклика после прогрева
  приемлемая (демон тёплый).
