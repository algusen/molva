# CONTRACT.md — Pisar API и форматы

Контракт между тонким клиентом (Finder Quick Action) и демоном (FastAPI),
а также спецификация выходных файлов и конфигурации. Любые правки контракта —
в этом файле, синхронно с кодом.

## Базовый URL

`http://127.0.0.1:<port>` — порт берётся из конфига, дефолт `8765`.
Демон **никогда** не слушает не-loopback интерфейсы.

---

## `GET /health`

Проверка готовности демона и состояния модели.

**Ответ `200 OK`:**
```json
{
  "status": "ready",
  "backend": "stub",
  "model_loaded": true,
  "version": "0.1.0"
}
```

`status`: `"loading"` | `"ready"` | `"error"`.
Пока `status != "ready"`, `/transcribe` отвечает `503`.

## `POST /transcribe`

**Тело запроса:**
```json
{
  "path": "/Users/me/Downloads/interview.m4a",
  "formats": ["txt", "srt"],
  "language": "ru"
}
```

- `path` — обязательный, абсолютный путь к файлу на диске. Файл не передаётся
  по сети, остаётся на месте.
- `formats` — список из `"txt" | "srt" | "vtt"`, минимум один. По умолчанию `["txt"]`.
- `language` — опциональный, ISO 639-1. По умолчанию `"ru"`. В Варианте 2 поддерживается
  только `"ru"`; другое значение → `400`.

**Ответ `200 OK`:**
```json
{
  "status": "done",
  "source": "/Users/me/Downloads/interview.m4a",
  "outputs": [
    "/Users/me/Downloads/interview.txt",
    "/Users/me/Downloads/interview.srt"
  ],
  "duration_sec": 187.4,
  "segments": [
    {"start": 0.0, "end": 4.2, "text": "Добрый день."}
  ]
}
```

Запросы обрабатываются строго последовательно (глобальный лок инференса):
второй параллельный запрос ждёт завершения первого, а не получает ошибку.

### Коды ошибок

| Код | Когда | Тело |
|-----|-------|------|
| `400` | неверный путь (не абсолютный), пустой `formats`, неподдерживаемый `language` | `{"error": "bad_request", "detail": "..."}` |
| `404` | файла нет по указанному пути | `{"error": "file_not_found", "detail": "..."}` |
| `415` | ffmpeg не распознаёт формат / не аудио-видео | `{"error": "unsupported_media", "detail": "..."}` |
| `422` | ffmpeg упал при препроцессинге | `{"error": "preprocessing_failed", "detail": "..."}` |
| `503` | модель ещё не загружена (`/health.status != "ready"`) | `{"error": "not_ready", "detail": "..."}` |
| `500` | непредвиденная ошибка инференса | `{"error": "internal_error", "detail": "..."}` |

Все ошибки — JSON с полями `error` (машиночитаемый код) и `detail` (для логов/уведомления).

## Sidecar-файлы: спецификация вывода

Для исходника `<dir>/<name>.<ext>` и формата `fmt` целевой путь —
`<dir>/<name>.<fmt>`.

**Правило конфликта имён:** если целевой файл уже существует и не был создан
этим же запуском, добавляется числовой суффикс: `<name> (1).<fmt>`,
`<name> (2).<fmt>` и т.д. — первый свободный. Существующие файлы пользователя
никогда не перезаписываются молча.

### `.txt`

Сплошной текст, сегменты соединены через `\n\n`. Без таймкодов.

### `.srt`

Стандартный SubRip:
```
1
00:00:00,000 --> 00:00:04,200
Добрый день.

2
00:00:04,200 --> 00:00:09,800
Сегодня поговорим о...
```
Таймкоды — `HH:MM:SS,mmm`, нумерация с 1, индексы по порядку сегментов.

### `.vtt`

WebVTT, тот же набор сегментов:
```
WEBVTT

00:00:00.000 --> 00:00:04.200
Добрый день.

00:00:04.200 --> 00:00:09.800
Сегодня поговорим о...
```
Таймкоды — `HH:MM:SS.mmm` (точка вместо запятой), без числовых индексов
(опционально, не обязательны по спеке WebVTT).

## Segment — внутренняя модель

```json
{"start": 0.0, "end": 4.2, "text": "Добрый день."}
```

- `start`, `end` — секунды от начала исходного файла (float), `end > start`.
- `text` — непустая строка, без ведущих/конечных пробелов.
- Список сегментов отсортирован по `start`, не должен иметь пересечений.

Это контракт между `Transcriber.transcribe()` и слоями `output.py`/API —
любая реализация транскрайбера (`StubTranscriber`, `GigaAMTranscriber`) обязана
возвращать `list[Segment]` в этом формате.

## Конфигурация демона

Источник: переменные окружения с префиксом `PISAR_`, либо файл
`~/.pisar/config.toml` (env приоритетнее файла). Дефолты — в `config.py`.

| Ключ | Env | Дефолт | Описание |
|------|-----|--------|----------|
| `host` | `PISAR_HOST` | `127.0.0.1` | bind-адрес, менять не рекомендуется |
| `port` | `PISAR_PORT` | `8765` | порт демона |
| `backend` | `PISAR_BACKEND` | `stub` | `stub` \| `gigaam` |
| `default_language` | `PISAR_LANGUAGE` | `ru` | язык по умолчанию |
| `default_formats` | `PISAR_FORMATS` | `txt` | CSV, напр. `txt,srt` |
| `log_dir` | `PISAR_LOG_DIR` | `~/Library/Logs/Pisar` | логи демона |
| `notify` | `PISAR_NOTIFY` | `true` | системные уведомления вкл/выкл |
| `clipboard` | `PISAR_CLIPBOARD` | `false` | копировать результат в буфер |

## Резерв под Вариант 3 (не реализуется в этом контракте сейчас)

Эти эндпоинты/поля зарезервированы по имени, чтобы не было конфликтов в будущем,
но **не реализуются** в Варианте 2:

- `POST /jobs` + `GET /jobs/{id}` — асинхронная постановка задачи, отдельно от
  синхронного `POST /transcribe`.
- `GET /jobs/{id}/events` (SSE) или WebSocket `/jobs/{id}/ws` — стриминг прогресса.
- `GET /history` — список прошлых транскрипций.
- `POST /diarize` — диаризация спикеров (pyannote), отдельный от `/transcribe` эндпоинт.
- Поле `language` в `/transcribe` зарезервировано под будущие значения помимо `"ru"`
  (точка расширения на Whisper), но сервер сейчас отклоняет всё, кроме `"ru"`.
