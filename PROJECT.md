# PROJECT.md — Pisar

## Что это

Pisar — локальный демон транскрибации аудио/видео на macOS (Apple Silicon),
на базе модели `ai-sage/GigaAM-v3`. Запускается правым кликом по файлу в Finder
(Quick Action), результат — sidecar-файлы (`.txt`/`.srt`/`.vtt`) рядом с исходником.

Это «Вариант 2» из общего плана развития: тёплый демон в памяти + тонкий клиент,
вызываемый из Finder. См. [PLAN.md](PLAN.md) для пошагового плана реализации
и фаз. Вариант 3 (SwiftUI-приложение) не реализуется сейчас, но API проектируется
с заделом под него.

## Цели

- Транскрибировать аудио/видео файл в текст с пунктуацией одним кликом из Finder.
- Держать модель тёплой в памяти между запросами (не грузить заново на каждый файл).
- Работать полностью локально — без сетевых вызовов вовне.
- Поддерживать длинные записи через нарезку по тишине (VAD), не теряя качество
  и не упираясь в лимиты контекста модели.
- Дать чистый HTTP-контракт, пригодный для повторного использования будущим
  нативным macOS-приложением (Вариант 3).

## Не-цели (явно вне рамок Варианта 2)

- Нет графического интерфейса/приложения с окном — только Finder Quick Action.
- Нет поддержки множества языков/моделей одновременно — только русский (GigaAM),
  точка расширения под Whisper заложена в API, но не реализуется.
- Нет стриминга прогресса, истории транскрипций, диаризации спикеров —
  зарезервировано в контракте, но не реализуется (Фаза 5).
- Нет multi-user / multi-tenant сценариев — однопользовательская локальная машина.
- Нет облачной обработки и удалённого хранения.
- Нет distribution через App Store / нотаризацию — установка через install.sh.

## Ограничения

- Платформа: macOS на Apple Silicon (для реального инференса GigaAM).
  Разработка и тесты ядра идут в Linux-контейнере на `StubTranscriber`.
- Инференс должен быть сериализован — один процесс держит одну модель в памяти,
  параллельные запросы становятся в очередь (риск OOM при параллельном инференсе).
- Демон слушает только `127.0.0.1` — никакого внешнего сетевого доступа.
- Зависимость от внешних бинарей: ffmpeg (обязательно), terminal-notifier
  (опционально, для уведомлений).
- Модель GigaAM-v3 грузится через `transformers` с `trust_remote_code=True` —
  ревизия `e2e_rnnt` фиксируется явно, чтобы не словить breaking change.

## Технологический стек

- Python 3.11+, окружение через `venv` (создаётся инсталлятором).
- FastAPI + uvicorn — HTTP-демон.
- ffmpeg — препроцессинг (извлечение/ресемплинг в 16 кГц mono WAV).
- silero-vad — нарезка длинных записей по паузам.
- GigaAM-v3 (`ai-sage/GigaAM-v3`, revision `e2e_rnnt`) через transformers + torch.
- launchd LaunchAgent — автозапуск/перезапуск демона.
- Finder Quick Action (`.workflow`-бандл) — точка входа пользователя.
- terminal-notifier / osascript — уведомления о завершении.

## Архитектурные принципы

- Транспорт — HTTP, по пути к файлу (не загрузка файла целиком).
- Конфигурируемый бэкенд транскрайбера (`stub` для тестов, `gigaam` для прода)
  через единый протокол `Transcriber`.
- Вывод — sidecar-файлы рядом с исходником, без перезаписи существующих
  (конфликт имён разрешается суффиксом).
- Контейнер используется для разработки и тестирования логики на заглушке;
  Mac — для интеграции с реальной моделью и системными механизмами macOS.
